# AI 使用人格测试

一个本地运行的 React + Tailwind CSS 网页应用，用来测试用户在使用 AI 时更偏向哪种协作方式。它不是心理诊断工具，而是一个认真但轻盈的 AI 互动风格测试。

## 当前状态

应用已经具备完整前端流程：

- 首页
- 量表答题页
- 情境题页
- 结果页
- 关于测试页

当前答题方式为单题展示：用户选择一道题的答案后，页面自动进入下一题；量表题答完后自动进入情境题，情境题答完后自动进入结果页。

## 初始状态

项目最开始只有：

- `README.md`
- `src/data/questions.js`
- `src/data/scenarios.js`

其中两个题库文件是固定题库来源。后续开发只允许读取这两个文件中的题目，不允许在页面代码中新增、改写或硬编码另一套题目。

## 数据来源

固定题库文件：

- `src/data/questions.js`
- `src/data/scenarios.js`

量表题要求：

- 共 48 道题
- 6 个维度
- 每个维度 8 道题
- 每个维度 6 道正向题、2 道反向题

情境题要求：

- 共 12 道题
- 每题 6 个选项
- A-F 分别对应 6 种 AI 使用人格倾向

如果题库数据缺失或结构不符合要求，应用会直接报错，不会自动补全题目。

## 抽题规则

每次进入测试时随机抽题：

- 从 48 道量表题中抽取 24 道
- 每个维度固定抽取 4 道
- 每个维度至少包含 1 道反向题
- 每个维度最多包含 2 道反向题
- 从 12 道情境题中抽取 3 道

## 计分规则

量表题使用 1-5 分：

- 1 = 非常不同意
- 2 = 比较不同意
- 3 = 一般
- 4 = 比较同意
- 5 = 非常同意

反向题按 `6 - 原始分` 计分。

每个维度原始分范围为 4-20 分，并转换为百分制：

```text
维度百分分 = (原始分 - 4) / 16 * 100
```

情境题不参与主分，只用于结果页补充说明。

## 结果内容

结果页展示：

- 主结果
- 结果清晰度
- 六个维度的百分制得分
- 雷达图
- 情境题选择最多的人格倾向
- 情境题与主结果的一致性提示
- 人格卡片
- 重新测试按钮
- 复制结果摘要按钮

## 技术栈

- React
- Vite
- Tailwind CSS
- 前端本地计算，无后端

## 本地运行

安装依赖：

```powershell
npm.cmd install
```

启动开发服务：

```powershell
npm.cmd run dev
```

访问：

```text
http://localhost:3000/
```

检查代码：

```powershell
npm.cmd run lint
```

构建生产版本：

```powershell
npm.cmd run build
```
